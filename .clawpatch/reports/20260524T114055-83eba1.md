## medium: Typecheck script can pass without running any type checks

id: fnd_sig-feat-release-a235aa99b1-80f0_c62e665bd2
category: build-release
confidence: medium
triage: risk
status: fixed
feature: Package script typecheck (feat_release_a235aa99b1)

evidence:
- package.json:27 (typecheck)
- package.json:34

The published root package advertises a `typecheck` command, but the command only recursively asks filtered child packages to run their own `typecheck` scripts and uses `--if-present`. That means the root command can exit successfully when selected packages have no `typecheck` script, and it does not directly invoke TypeScript despite TypeScript being a dev dependency. This can make CI/release validation report success while no package was actually typechecked.

recommendation:
Make the root `typecheck` script fail unless the intended TypeScript checks actually run, for example by invoking the concrete `tsc --noEmit` commands or removing `--if-present` once every package is expected to define `typecheck`.

test analysis:
No linked tests are included for the package script behavior, and package scripts are not validated by the shown package metadata.

suggested regression test:
Add a packaging or CI smoke test that runs `pnpm typecheck` and asserts the expected TypeScript projects are checked rather than silently skipped.

minimum fix scope:
Update the root `typecheck` script and, if needed, add/standardize package-level `typecheck` scripts.
